1 Executive Summary

วิเคราะห์ร้านอาหารสำหรับมื้อทีม 8–12 คน ใน 5 ย่าน ด้วยข้อมูลจริงจาก Apify / Google Maps + แหล่งข้อมูลที่ 2 (Wongnai / Tripadvisor) ให้คะแนนด้วย Scoring Model 100 คะแนน แล้วคัด Top 10 และ Top 3.

2 Top 3 Recommendation

3 อันดับแรกจาก Top 10 — ตัวเลือกที่แนะนำสำหรับมื้อทีม (ข้อมูลและคะแนนจริงจาก dataset)

6 Scoring Criteria (100 คะแนน)

น้ำหนักคงที่ — ทุกร้านถูกให้คะแนนด้วยเกณฑ์เดียวกันจากข้อมูลจริง

#เกณฑ์น้ำหนักวัดจากอะไรสัดส่วน
1Rating & Review Quality25ดาวเรตติ้ง + ขนาดฐานรีวิว (ความน่าเชื่อถือ)
2Group Suitability20รองรับโต๊ะ 8–12 คน / มีพื้นที่กลุ่ม
3Price Suitability15ราคาต่อหัวเหมาะกับมื้อทีม (mid-range ได้เปรียบ)
4Travel Convenience15ระยะเดินจาก BTS/MRT ที่ใกล้ที่สุด
5Data Completeness15ความครบ/ความมั่นใจของข้อมูล (จุดที่ต้องยืนยัน)
6Uniqueness / Experience10เอกลักษณ์/ประสบการณ์ (Michelin, rooftop, concept)
Total100ผลรวมทั้ง 6 มิติ — จัดอันดับจากมากไปน้อย

7 Workflow Overview

Pipeline จากการตั้งโจทย์ → เก็บข้อมูล → ทำความสะอาด → ให้คะแนน → automation → รายงาน

1

Problem Framing

นิยามโจทย์ทีม 8–12 คน, 5 ย่าน, เกณฑ์ตัดสินใจ

2

Apify Data Scraping

ดึงข้อมูลร้านจาก Google Maps (rating, รีวิว, พิกัด)

3

Google Sheets DB

เก็บ Raw / Clean / Scoring / Top 10 / Top 3

4

AI Cleaning & Enrichment

Claude/ChatGPT เติมราคา เวลา group-fit + ยืนยันแหล่งที่ 2

5

Scoring Model

ให้คะแนน 100 คะแนน 6 มิติ จัดอันดับ

6

n8n Automation

Connected production webhook: HTML Ask Box → n8n Webhook → Fetch Top 10 CSV → Fetch Top 3 CSV → Code JS → Respond to Webhook

7

n8n Live Data API

Live data sync: HTML เรียก n8n Production Webhook เพื่อโหลด restaurantsData, top10 และ top3 จาก Google Sheets CSV แล้ว render Top 3, Top 10, Area Filter และ Restaurant Comparison

8

HTML Report

รายงานหน้านี้ — ตัดสินใจได้จริง ตรวจย้อนกลับได้

11 Notes / Limitations

  • ข้อมูลบางส่วน เช่น ราคา เวลาเปิด–ปิด การเดินทาง และการรองรับโต๊ะ 8–12 คน อาจต้องยืนยันกับร้านก่อนจองจริง
  • Final decision ควรตรวจสอบ เวลาเปิด–ปิด ราคา และการจองโต๊ะ กับร้านอีกครั้งก่อนยืนยัน
  • คะแนนคำนวณด้วย Scoring Model 100 คะแนน จากชุดข้อมูลที่ clean แล้ว โดยใช้เกณฑ์เดียวกันทุกร้าน
  • n8n Live Data API ใช้สำหรับดึงข้อมูลจาก Google Sheets CSV 3 ชุด ได้แก่ Scoring, Top 10 Ranking และ Top 3 Recommendation เข้าสู่ HTML report
  • HTML report ใช้ live data จาก n8n เป็นหลัก และใช้ embedded clean dataset เป็น fallback สำหรับรายละเอียดบางช่องที่ไม่ได้อยู่ใน CSV live
  • Dataset Ask Box ทำงานผ่าน n8n Production Webhook โดยตอบจากข้อมูลใน Google Sheets CSV เท่านั้น ไม่ใช้ความรู้ทั่วไป
